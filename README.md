
# MediTrack Connect - 全院中心化数据库管理工具

本项目是一个专为医疗机构设计的中心化数据库管理前端工具。采用 C/S 架构，通过 Electron 技术封装，实现跨终端的远程 MySQL 数据库实时同步与附件共享管理。

---

## 一 系统架构与核心定位

1. **远程数据库核心**：本程序不再区分服务器端与客户端程序，所有终端均连接至同一台远程中心 MySQL 服务器。
2. **零配置启动**：程序启动后通过登录页底部的“服务器连接设置”随时校准数据库接入参数。
3. **中心化存储**：PDF 报告及影像扫描件由全院共享的物理存储路径（UNC 路径）统一管控。

---

## 二 环境准备与配置要求

### 1 数据库服务配置 (中心端)
- **推荐引擎**：MySQL 8.0.x
- **监听端口**：默认 `10699`
- **初始化脚本**：请确保数据库用户具有完整的 `ALL PRIVILEGES` 权限，系统首次连接时会自动同步 7 张业务表。

### 2 附件存储配置 (全院统一)
- **物理路径**：建议在中心存储服务器创建共享文件夹。
- **引用标准**：在系统设置中填入 UNC 路径（如 `\\SERVER_IP\MediStorage`）。
- **读写权限**：确保运行该程序的 Windows 账户对该路径具有完全控制权限。

---

## 三 核心业务表结构

系统自动同步以下 7 张核心表，确保护理与随访数据的闭环管理：

- **SP_USERS**: 人员权限与 Admin 安全凭据表。
- **SP_SETTINGS**: 全院全局配置表（包含系统名称及 STORAGE_PATH）。
- **SP_PERSON**: 患者电子档案核心主表。
- **SP_ZYJG**: 重要异常结果登记流水。存储 14 维登记信息。
- **SP_SF**: 随访闭环结案流水。存储 9 维回访结论及复查标记。
- **SP_LOGS**: 全量操作审计日志。
- **SP_DOCUMENTS**: 附件索引表，关联物理文件路径。

---

## 四 客户端部署与加固

### 1 安装前提
- **运行库**：必须安装 **Microsoft Visual C++ Redistributable 2015-2022**。
- **兼容性加固**：程序已针对 Windows 7/8.1 禁用硬件加速及沙盒模式，解决“启动无反应”问题。

### 2 打包流程
1. `npm install`
2. `npm run dist`
打包产物位于 `dist` 目录。

### 3 运行诊断
- **日志分析**：程序自动记录 SQL 报错与系统轨迹至 `%APPDATA%/meditrack-connect/app.log`。
