
# 重要异常结果管理系统 - 全院中心化数据库管理工具

本项目是一个专为医疗机构设计的 **中心化、纯数据库驱动** 的异常结果闭环管理工具，深度适配 MySQL 8.4 (Winx64) 服务器环境。

---

## 一 系统架构与核心定位

1. **远程数据库驱动**：所有终端直接接入同一台远程 MySQL 8.4 服务器，实现全院数据多点并发、实时同步。
2. **双重周期预警引擎**：
   - **初次随访 (T+7)**：基于“通知日期”自动计算。
   - **年度复查 (T+365)**：基于“体检流水号”前 8 位自动物理解析出的体检日期进行计算。
   - **智能任务闭环**：若在 T+7 周期内已完成随访结案，系统会自动物理停办该提醒任务，避免重复劳动。
3. **两步式临床采集流**：
   - 实现“重要异常结果登记 -> 患者基础资料完善”的无缝转场，极大提升了临床数据采集的完整性。
   - 支持“跳过”逻辑，允许后续在档案中心进行资料补录。
4. **全时序电子档案**：
   - 个人档案页面以统一时间轴展示所有临床轨迹（异常记录与随访流水）。
   - **上下文附件关联**：系统根据日期自动将 PDF 报告关联至对应的业务节点，实现“病历与报告合一”。

---

## 二 管理员中心配置

### 1 动态临床集成 (PACS)
- **配置路径**：管理员可在“全院管理中心”配置 PACS 影像浏览器跳转地址模板。
- **动态占位符**：支持 `${id}` 占位符，调阅时自动替换为患者档案编号并唤醒本机浏览器。

### 2 视觉资产与中心存储 (UNC)
- **资产同步**：系统 Logo 及登录背景物理同步至中心存储库，全院客户端自动缓存。
- **物理分拣逻辑**：文件自动按 `[患者ID] / [报告类别] / [日期_文件名]` 逻辑进行物理分拣存储。

---

## 三 稳定性与易用性加固

1. **提交防抖锁定**：所有业务提交按钮（登记、上传、配置）均集成物理锁定机制，点击后立即禁用并展示 Loading，物理隔绝并发重复点击。
2. **多主题支持**：内置普通医疗蓝、柔和护眼、深海夜间三种主题模式，适配不同临床工作光照环境。
3. **性能兼容性**：针对医疗终端（Win 7/10/11），程序默认强制执行 GPU 硬件加速禁用策略，确保低配机型运行流畅。

---

## 四 核心业务表结构

- **SP_USERS**: 人员权限表，管理操作员凭据。
- **SP_SETTINGS**: 全局配置，包含 PACS 模板及存储路径。
- **SP_PERSON**: 电子病历主表，支持 `ON DUPLICATE KEY UPDATE` 物理去重。
- **SP_ZYJG**: 重要异常结果流水，包含宣教、反馈及处置意见。
- **SP_SF**: 随访闭环结案流水。
- **SP_DOCUMENTS**: 附件索引表，与业务流通过日期物理关联。
