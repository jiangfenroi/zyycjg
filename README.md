
# MediTrack Connect - 医疗闭环管理系统部署与使用手册 (v1.7)

本项目是一个专为医疗机构设计的 **C/S (Client/Server) 架构** 闭环管理系统。采用 Next.js 静态导出技术并嵌入 Electron 容器，后端接入中心化 MySQL 数据库，用于重要异常结果的登记、随访、预警与报表导出。

---

## 一、 打包流程 (开发环境)

**注意：** 配置信息（如数据库 IP、密码等）**不需要**在打包前设置。程序采用动态接入向导设计，打包生成的安装包可分发到全院任何终端。

### 1. 环境准备
- 安装 **Node.js v18** 或更高版本。
- 准备一台 **MySQL 8.0** 以上版本的中心服务器（确保 3306 端口在局域网内可访问）。

### 2. 打包命令
1. **安装依赖**: 
   ```bash
   npm install
   ```
2. **生成静态资源**:
   ```bash
   npm run build
   ```
   执行后会生成 `out` 目录，这是 Next.js 导出的纯静态 HTML/JS 文件。
3. **构建可执行程序**:
   ```bash
   npm run dist
   ```
   执行后在 `dist/` 目录下会生成 `MediTrack Connect Setup.exe`。

## 二、 安装与接入 (生产环境)

### 1. 客户端安装
在目标电脑运行 `MediTrack Connect Setup.exe`，按照向导完成安装。安装程序会自动创建桌面快捷方式。

### 2. 首次运行配置 (接入向导)
启动程序后，系统会执行以下逻辑：
- **客户端首次运行判定**：系统检测本地用户目录 `%APPDATA%/meditrack-connect/` 下是否存在配置文件 `db-config.json`。
- **进入向导**：若不存在配置文件，系统自动跳转至“中心服务器接入”页面。
- **填写配置**：
  - **服务器主机**: 输入 MySQL 中心服务器的 IP 地址。
  - **端口**: 默认为 3306。
  - **访问账号/密码**: 用于连接 MySQL 的凭据。
  - **数据库名称**: 默认为 `meditrack_db`（若服务器上不存在该库，系统将尝试自动创建）。

### 3. 服务器端自动初始化
点击“建立连接并初始化中心端”后，程序将：
1. 在远程服务器上创建指定的数据库 Schema（如果不存在）。
2. **自动创建业务表**：自动检测并创建所有 8 张业务表（包含用户表、异常结果表、随访记录表等）。
3. **初始化管理员**：自动创建默认账号 `admin`，初始密码 `123456`。

## 三、 核心业务维度标准

### 1. 重要异常结果登记 (16 维)
系统登记界面及导出报表严格包含以下 16 项关键维度：
1. 档案编号
2. 体检编号
3. 姓名
4. 性别
5. 年龄
6. 联系电话
7. 体检日期
8. 结果分类 (A/B)
9. 重要异常结果详情
10. 是否通知
11. 是否健康宣教
12. 通知日期
13. 通知时间
14. 通知医生 (登记人)
15. 被通知人 (患者/家属)
16. 处置建议

### 2. 已回访记录导出 (11 维)
随访管理模块导出的 CSV/Excel 表格包含以下 11 项业务维度：
1. 档案编号
2. 体检编号
3. 姓名
4. 性别
5. 年龄
6. 重要异常结果详情
7. 回访结果详情
8. 是否复查及进一步病理检查
9. 回访时间
10. 回访人
11. 下次回访时间 (闭环标识)

## 四、 关键功能说明

- **静默预警**: 
  - 开启“开机自启动”后，程序开机将静默运行于系统托盘。
  - 主进程每 5 分钟扫描一次中心数据库。
  - 发现待随访人员时，自动触发 **Windows 系统通知** 并在 **任务栏闪烁** 图标。
- **报表导出**: 
  - 所有导出功能均增加 UTF-8 BOM 标记，确保生成的 CSV 文件在 Excel 中直接打开不乱码。
- **自动恢复与容错**: 
  - 若中心服务器宕机或网络中断，客户端所有业务操作将拦截并提示错误。
  - 系统会实时监测数据库心跳，若连接失效，将自动引导用户返回接入向导页面重新检查配置。
