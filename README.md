
# MediTrack Connect - 医疗闭环管理系统部署与使用手册 (最终版)

本项目是一个专为医疗机构设计的 **C/S (Client/Server) 架构** 闭环管理系统。采用 Next.js 静态导出技术并嵌入 Electron 容器，后端接入中心化 MySQL 数据库，用于重要异常结果的登记、随访、预警与报表导出。

---

## 一、 打包流程 (开发环境)

**核心原则：** 数据库配置信息 **不需要** 在打包前设置。打包生成的安装包是通用的，所有的接入参数均在安装后通过客户端向导进行动态配置。

### 1. 环境准备
- 安装 **Node.js v18** 或更高版本。
- 准备一台 **MySQL 8.0** 以上版本的中心服务器。
- **重要：** Windows 8.1/7 用户请确保安装了 **Microsoft Visual C++ Redistributable** 运行库，否则数据库模块可能无法初始化。

### 2. 打包命令
1. **安装依赖**: 
   ```bash
   npm install
   ```
2. **静态资源构建**:
   ```bash
   npm run build
   ```
   执行后会生成 `out` 目录，这是 Next.js 导出的纯静态文件，已适配 Electron 协议。
3. **生成安装程序**:
   ```bash
   npm run dist
   ```
   执行后在 `dist/` 目录下会生成 `MediTrack Connect Setup.exe`。

## 二、 安装与接入 (生产环境)

### 1. 客户端安装
在目标电脑运行 `MediTrack Connect Setup.exe`。安装程序会自动创建桌面快捷方式。
**Win 8.1 兼容性说明：** 若程序运行无反应，系统会自动禁用硬件加速并尝试重新启动，请确保显卡驱动已更新。

### 2. 首次运行配置 (网络版接入向导)
启动程序后，系统会自动判断：
- **客户端侧判定**：若本地无 `db-config.json`，则自动进入“中心服务器接入”页面。
- **填写配置**：
  - **服务器主机**: 输入 MySQL 服务器的 IP 地址。
  - **端口**: 3306。
  - **访问账号/密码**: MySQL 访问凭据。
  - **数据库名称**: 留空则默认使用 `meditrack_db`。
- **服务器侧初始化**：点击“建立连接并初始化中心端”后，程序会自动连接服务器并执行以下逻辑：
  1. 自动创建 Schema。
  2. 自动创建所有业务表。
  3. 自动创建默认管理员账号 `admin`，初始密码 `123456`。

### 3. 多终端同步
在全院其他电脑安装后，只需输入同一台 MySQL 服务器的 IP，所有终端将实时同步患者档案、登记记录及随访状态。

## 三、 业务维度标准

### 1. 重要异常结果登记标准 (16 维)
系统登记界面及导出报表严格包含以下维度：
1. 档案编号
2. 体检编号
3. 姓名
4. 性别
5. 年龄
6. 联系电话
7. 体检日期
8. 结果分类
9. 重要异常结果详情
10. 是否通知
11. 是否健康宣教
12. 通知日期
13. 通知时间
14. 通知医生
15. 被通知人
16. 处置建议

### 2. 重要异常随访结案标准 (11 维)
随访管理模块导出的记录包含以下维度：
1. 档案编号
2. 体检编号
3. 姓名
4. 性别
5. 年龄
6. 重要异常结果详情
7. 回访结果详情
8. 是否复查及进一步病理检查
9. 回访时间
10. 回访人
11. 下次回访时间

## 四、 关键功能说明

- **开机自启动**: 程序可在设置中开启开机启动，启动后静默运行于系统托盘，不干扰日常办公。
- **智能预警**: 
  - 任务栏图标闪烁：发现待随访人员时，图标会实时闪烁。
  - 系统通知：后台静默扫描中心数据库，发现任务时直接弹出 Windows 系统通知。
- **断连重定向**: 若中心服务器宕机，客户端会自动跳转至配置页面，防止数据丢失。
- **附件共享**: 建议配合医院 UNC 共享路径存储病历附件，实现全院实时查阅。
