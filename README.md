
# MediTrack Connect - 医疗闭环管理系统部署手册 (v1.2 最终版)

本项目是一个专门为医疗机构设计的 **Client-Server (C/S)** 架构管理系统，旨在实现重要异常结果（A/B类）的从发现、登记、通知到随访结案的全闭环流程管理。系统原生适配 Windows 7 / 8.1 / 10 / 11 及其以上系统。

## 一、 系统环境要求

### 1. 核心兼容性 (Legacy Windows Support)
- **客户端支持**: Windows 7 (SP1+), Windows 8.1, Windows 10/11 (x64)
  - *系统内核锁定至 Electron 22.x 系列，确保在旧版 Windows 上调用原生 API 的稳定性。*
- **服务器端**: Windows Server 2012+ (推荐) 或 Windows 10/11
- **数据库**: MySQL 8.0+

### 2. 中心存储配置 (UNC 网络共享)
为了实现“一处上传，全院查看”，必须配置 Windows 共享文件夹：
1. 在服务器上创建一个文件夹（如 `D:\meditrack_storage`）。
2. 将该文件夹设置为 **Windows 共享文件夹**。
3. 确保各终端能通过 UNC 路径（如 `\\服务器IP\共享名`）直接读写。
4. 打包前在 `.env` 中设置 `UPLOAD_PATH=\\服务器IP\共享名`。

## 二、 核心业务维度标准

### 1. 重要异常结果登记 (16 维标准)
系统表格展示及导出的 Excel 报表严格包含以下 16 项核心维度：
- 档案编号、体检编号、姓名、性别、年龄、联系电话、体检日期、分类、重要异常结果详情、是否通知、是否健康宣教、通知日期、通知时间、通知医生、被通知人、处置建议。

### 2. 重要异常结果随访 (闭环标准)
- **待随访任务 (6 维)**: 体检编号、姓名、电话、异常结果详情、通知日期、处置建议。
- **已回访记录 (11 维)**: 档案编号、体检编号、姓名、性别、年龄、异常结果详情、回访结果详情、是否复查及进一步病理检查、回访时间、回访人、下次回访时间。

## 三、 数据库部署 (MySQL)

### 1. 初始化脚本
连接至 MySQL 服务器并执行：
```sql
CREATE DATABASE meditrack_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 创建远程访问账号
CREATE USER 'medi_admin'@'%' IDENTIFIED BY 'AdminPassword123';
GRANT ALL PRIVILEGES ON meditrack_db.* TO 'medi_admin'@'%';
FLUSH PRIVILEGES;
```

### 2. 核心业务表结构
系统首次连接成功后会自动创建/更新以下核心表：
- `SP_USERS`: 账户角色与工号管理。
- `SP_PERSON`: 全院患者基础档案库。
- `SP_ZYJG`: 重要异常结果登记表（16 维数据源）。
- `SP_SF`: 随访沟通记录与结案流水（含下次回访日期）。
- `SP_SFRW`: 计划复查任务表。
- `SP_DOCUMENTS`: PDF 报告与影像扫描件路径索引。
- `SP_SETTINGS`: 全局品牌标识。
- `SP_LOGS`: 全量操作审计日志。

## 四、 关键功能说明

1. **预警系统**: 铃铛图标通知中心会实时预警全院范围内**未随访的 A 类和 B 类**重要异常结果。
2. **数据导出**: 支持将登记列表和已回访列表一键导出为 Excel 兼容的 CSV 表格。
3. **附件预览**: 采用 `app-file` 安全协议，支持直接在客户端内预览服务器共享路径下的 PDF 报告。
4. **UI 规范**: 界面经过深度净化，删除了所有技术性括号内容，确保医疗文书展示的严谨性。
