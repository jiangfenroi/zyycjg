
# 重要异常结果管理系统 - 全院中心化数据库管理工具

本项目是一个专为医疗机构设计的 **中心化、纯数据库驱动** 的异常结果闭环管理工具。

---

## 一 系统架构与核心定位

1. **远程数据库驱动**：所有终端直接接入同一台远程 MySQL 服务器，实现全院数据多点并发、实时同步。
2. **资产同步引擎**：系统图标、Logo、登录页背景均由管理员统一设置，并物理同步至中心存储库，全院客户端自动缓存并同步显示。
3. **物理分拣逻辑**：文件自动按 `[患者ID] / [报告类别] / [日期_文件名]` 逻辑进行物理分拣，确保海量病历附件的有序组织。
4. **实时分析工作台**：自动化数据统计引擎，实时聚合档案总量、闭环完成率及月度随访趋势。

---

## 二 管理员中心配置

### 1 视觉资产配置
- 管理员可在系统设置中上传系统 Logo 及登录背景。
- 图片将自动物理拷贝至 `STORAGE_PATH/system_assets/` 目录下，并全局分发路径。

### 2 中心附件存储 (UNC 共享)
- **配置路径**：管理员配置 UNC 路径（如 `\\172.17.16.18\MediStorage`）。
- **权限要求**：确保运行账户具有读/写权限。

---

## 三 核心业务表结构

- **SP_USERS**: 人员权限表。存储操作员凭据。
- **SP_SETTINGS**: 全局配置。包含品牌资产及存储路径。
- **SP_PERSON**: 电子病历主表。
- **SP_ZYJG**: 重要异常结果流水。
- **SP_SF**: 随访闭环结案流水。
- **SP_DOCUMENTS**: 附件索引表。

---

## 四 兼容性与加固

针对 Windows 7/8.1，程序默认强制执行以下策略：
- 禁用 GPU 硬件加速
- 禁用沙盒模式
- 禁用软件光栅化渲染
