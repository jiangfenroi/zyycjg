# MediTrack Connect - 医疗闭环管理系统部署与使用手册 (v1.5)

本项目是一个专为医疗机构设计的 **Client-Server (C/S)** 架构管理系统。系统支持 Windows 7 / 8.1 / 10 / 11，核心用于重要异常结果（A/B类）的登记、随访与闭环管理。

---

## 一、 打包前环境准备

1. **Node.js**: 推荐 v18+。
2. **MySQL 服务器**: 确保医院中心服务器已安装 MySQL 8.0+。
3. **共享文件夹**: 
   - 在服务器上创建文件夹（如 `D:\meditrack_storage`）。
   - 设置为 **Windows 共享文件夹**，确保各终端拥有读写权限。
   - 记录 UNC 路径（如 `\\192.168.1.100\meditrack_storage`）。

---

## 二、 程序打包流程 (开发者/管理员操作)

本系统采用 Next.js 静态导出并嵌入 Electron 的方案。

1. **安装依赖**:
   ```bash
   npm install
   ```

2. **执行静态导出**:
   ```bash
   npm run build
   ```
   *此命令会生成 `out` 文件夹，包含所有静态 HTML/JS/CSS。*

3. **执行打包程序**:
   ```bash
   npm run dist
   ```
   *此命令使用 `electron-builder` 生成 Windows 安装包。*
   *生成的安装文件位于 `dist/MediTrack Connect Setup.exe`。*

---

## 三、 安装与首次配置 (终端用户操作)

### 1. 安装
运行 `MediTrack Connect Setup.exe`，按照向导完成安装。程序将自动在桌面创建快捷方式。

### 2. 数据库配置 (核心步骤)
**注意：配置信息是在打包后运行程序时设置的，而非打包前。**

1. **首次启动**: 程序会检测到未连接数据库，自动进入 **“网络版接入向导”**。
2. **输入参数**:
   - **服务器主机**: 输入医院中心数据库的 IP 地址。
   - **访问账号/密码**: 输入由医院网管提供的 MySQL 账号。
   - **数据库名称**: 默认为 `meditrack_db`。
3. **完成接入**: 点击“测试并接入中心网络”，程序会自动创建所需的业务表并跳转至登录页面。

*配置信息将加密保存在本地：`%APPDATA%/meditrack-connect/db-config.json`，后续启动无需重复输入。*

## 四、 核心业务维度标准

### 1. 重要异常结果登记 (16 维)
包含：档案编号、体检编号、姓名、性别、年龄、联系电话、体检日期、分类、重要异常结果详情、是否通知、是否健康宣教、通知日期、通知时间、通知医生、被通知人、处置建议。

### 2. 重要异常结果随访 (闭环标准)
- **待随访任务 (6 维)**: 体检编号、姓名、电话、异常结果详情、通知日期、处置建议。
- **已回访记录 (11 维)**: 档案编号、体检编号、姓名、性别、年龄、异常结果详情、回访结果详情、是否复查及进一步病理检查、回访时间、回访人、下次回访时间。

---

## 五、 故障排除

1. **启动卡在“系统正在连接...”**: 
   - 请检查终端是否能 Ping 通中心服务器 IP。
   - 检查中心服务器 MySQL 是否允许远程连接（%）。
2. **资源加载失败 (ERR_FILE_NOT_FOUND)**:
   - 确保使用的是 `npm run build` 生成的最新 `out` 目录进行打包。
   - 系统已内置 `app://` 协议自动修复路径问题。
3. **无法预览 PDF 报告**:
   - 确保各终端已正确挂载或可访问服务器的 UNC 共享路径。
