
# MediTrack Connect - 医疗闭环管理系统部署手册 (v1.6)

本项目是一个专为医疗机构设计的 **C/S 架构** 闭环管理系统，用于重要异常结果的登记、随访与报表导出。

---

## 一、 打包流程 (开发环境)

系统采用 Next.js 静态导出并嵌入 Electron。配置信息不需要在打包前设置。

1. **环境准备**: 安装 Node.js v18+ 及 MySQL 8.0+ 服务器。
2. **安装依赖**: `npm install`
3. **导出静态资源**: `npm run build` (生成 `out` 目录)
4. **生成安装包**: `npm run dist` (生成 `dist/MediTrack Connect Setup.exe`)

---

## 二、 安装与接入 (生产环境)

**配置信息是在打包后、首次运行程序时通过向导设置的。**

### 1. 客户端安装
运行 `MediTrack Connect Setup.exe`。

### 2. 接入中心服务器 (首次运行)
- 启动程序，系统将检测到未连接数据库，自动弹出 **“中心服务器接入”** 向导。
- 输入中心 MySQL 服务器的 IP、端口、账号、密码及数据库名称（Schema）。
- 点击“建立连接并初始化中心端”，系统将自动：
  - 在服务器上创建数据库（如果不存在）。
  - 自动创建所有业务表。
  - 初始化默认管理员账号：`admin` / `123456`。

---

## 三、 核心业务维度标准

### 1. 重要异常结果登记 (16 维)
包含：档案编号、体检编号、姓名、性别、年龄、联系电话、体检日期、分类、重要异常结果详情、是否通知、是否健康宣教、通知日期、通知时间、通知医生、被通知人、处置建议。

### 2. 已回访记录导出 (11 维)
包含：档案编号、体检编号、姓名、性别、年龄、重要异常结果详情、回访结果详情、是否复查及进一步病理检查、回访时间、回访人、下次回访时间。

---

## 四、 关键功能说明

- **静默预警**: 开启“开机自启动”后，程序将最小化到系统托盘。每隔 5 分钟自动扫描数据库，发现待随访人员将通过 Windows 系统通知并闪烁任务栏。
- **报表导出**: 支持将登记数据和随访数据直接导出为符合 Excel 标准的 CSV 表格。
- **自动恢复**: 若中心服务器连接中断，客户端将自动跳转回接入向导页面，确保数据一致性。
